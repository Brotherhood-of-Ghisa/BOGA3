# t6: cycle implementation per t2 §6

**Type:** build

**Problem:** The v2 cycle is the heart of the client sync engine. It
converges local + remote state by issuing PULL → PUSH → PULL until
both ends are quiet, with the wire-shape and batching rules pinned
by t2 §3 (push), §4 (pull), §4.4 (layered drain), §3.4 (push batch
selection), §6 (cycle interface), and §7 (dirty-bit lifecycle). It
runs inside a single async function with no surviving in-memory
state per t2 §6.2; reconstructs from SQLite on every call;
idempotent on re-run per t2 §6.2; handles the push-in-flight race
per t2 §7.3; and treats the AUTH_REQUIRED envelope as a normal
error envelope per t2 §2.2 (not a network failure).

**Inputs:**

- t2 merged (schemas + cursor column).
- t5a + t5b merged (every dirty stream entry is genuine — every
  repo write flips the dirty bit).
- `docs/plans/sync-v2/designs/t2.md` §3, §4, §4.4, §6, §7 — the
  binding contract for everything in this task.
- `docs/plans/sync-v2-server/plan.md ## Deviations log` — RPC
  signatures (t3 entry, named param `entities`; t4 entry, unnamed
  jsonb param).
- `apps/mobile/src/sync/topo-order.ts` — Layer 0..3 partition.
  Imported, not redefined.
- `apps/mobile/src/data/clock.ts` — `nowMonotonic` (t4 — not used
  inside the cycle directly except where the cycle writes a row,
  e.g. pull-apply, which uses the incoming `client_updated_at_ms`
  per t2 §4.5).
- Supabase client: existing `@supabase/supabase-js` instance under
  `apps/mobile/src/lib/` or `apps/mobile/src/auth/` (grep before
  starting).

**Outcomes:**

- A new file `apps/mobile/src/sync/cycle.ts` exports:
  - `runSyncCycle(): Promise<void>` — the entry point. Returns
    when the cycle has converged or aborted with progress (t2
    §6.2). Throws on `FK_VIOLATION` (non-retriable). Returns
    cleanly on `AUTH_REQUIRED` / `INTERNAL` (the scheduler in t7
    re-arms LONG_INTERVAL).
  - Internal-but-exported helpers for unit testing:
    `selectPushBatch(tx, batchCap)`, `applyPullPage(tx, entities,
    layer)`, `entityToWire(row, type)`, `wireToEntity(envelope,
    type)`. The exact list is the builder's call; the tests in
    the Outcomes block need access to each piece.
- **Push leg** per t2 §3:
  - Calls `selectPushBatch(batchCap = 200)` per t2 §3.4.2 inside a
    SQLite transaction; the algorithm walks layers in topological
    order from `TOPO_LAYERS`.
  - Captures `Map<(type, id), sent_at_ms>` in cycle-local memory
    per t2 §7.3 BEFORE the HTTP request.
  - POSTs `{"entities": [...]}` to `/rest/v1/rpc/sync_push`
    (named param `entities` per plan 1's t3 Deviations log entry).
  - On success ack, iterates the captured map: for each row,
    inside a fresh SQLite transaction, SELECT the current
    `local_updated_at_ms`; if `== sent_at_ms`, clear `local_dirty
    = 0` (one transaction per row per t2 §7.3, so a concurrent
    edit doesn't get clobbered).
  - On `FK_VIOLATION`: throw — non-retriable per t2 §2.2.
  - On `AUTH_REQUIRED`: return cleanly (the cycle gives up;
    dirty bits stay set; the scheduler will re-tick).
  - On `INTERNAL` / transport error: return cleanly with the
    same disposition.
  - Multi-batch push: when `selectPushBatch` returns a non-empty
    batch and the cycle's elapsed time is under budget (see t2
    §6.2 — 20s background / 5s foreground soft), loop.
- **Pull leg** per t2 §4 / §4.4:
  - For each layer in `[0, 1, 2, 3]`, drains per t2 §4.4's
    pseudocode: read `cursors[layer]` from
    `sync_runtime_state.pull_cursor`, POST `{"layer": N, "cursor":
    ..., "limit": 200}` to `/rest/v1/rpc/sync_pull` (unnamed jsonb
    per plan 1's t4 Deviations log entry), apply the page per t2
    §4.5 (per-row LWW), advance `cursors[layer]` after COMMIT
    (t2 §4.7).
  - `apply page` is one SQLite transaction containing every row
    in the response. Per-row LWW per §4.5: SELECT by `(type,
    id)`; INSERT or UPDATE per the four cases; never reapply
    deletes blindly (every column from `fields` is overwritten —
    `deleted_at` is one of them).
  - `pull_cursor` updates: read as JSON, update the per-layer
    entry, write back via Drizzle's `mode: 'json'`. Persisted in
    the SAME transaction as the page apply.
  - Loop within a layer until `has_more === false`; then advance
    to the next layer.
  - On `AUTH_REQUIRED` / `FK_VIOLATION` / `INTERNAL`: same posture
    as push (FK_VIOLATION throws, others return cleanly with
    cursor unchanged for the affected layer).
- **Cycle shape** per t2 §6.1:
  - PULL (drain all 4 layers) → PUSH (drain dirty stream) → PULL
    (drain all 4 layers).
  - Convergence: repeat until both ends are quiet for one full
    round. Concrete loop in code; not an infinite spin (a
    `MAX_CYCLES_PER_CALL = 5` constant prevents pathological
    runaway from clock-skew-induced LWW thrash).
- **Wire serialisation** per t2 §2.1 / §3.3 / §9.5:
  - `entityToWire(row, type)` emits every server-typed column from
    `fields` (per t1 §2 mapping); omits `local_dirty` and
    `local_updated_at_ms`; emits `deleted_at` as a normal column
    under LWW.
  - For `gyms`, the four M15 carry-over columns (`latitude`,
    `longitude`, `coordinate_accuracy_m`, `coordinates_updated_at`)
    are emitted per `## Carry-over` item 2.
  - The wire envelope uses `client_updated_at_ms =
    row.local_updated_at_ms`.
- Unit tests cover:
  - `selectPushBatch`: walks layers in order; respects `batchCap`;
    breaks out of the loop when the cap is hit; intra-layer
    FIFO by `local_updated_at_ms ASC`.
  - `applyPullPage`: per-row LWW (cases 2 / 3 / 4 from §4.5); the
    page-rollback path on FK violation; cursor advances after
    COMMIT.
  - `entityToWire` / `wireToEntity` round-trip for each of the
    eight entity types.
  - Push-in-flight race (t2 §7.3): inject a concurrent local
    edit between batch capture and ack; assert the dirty bit
    stays `1`.
  - `AUTH_REQUIRED` envelope handling: stub the RPC to return
    the error shape from t2 §2.2; assert the cycle returns
    cleanly with no SQLite mutation.
  - Convergence: stub a server that returns one new row on the
    first pull and is empty on the third pull; assert the cycle
    terminates with two rounds of PULL → PUSH → PULL.
- `npm run lint && npm run typecheck && npm run test` passes from
  `apps/mobile/`.

**Output artifact:**

- New file `apps/mobile/src/sync/cycle.ts`.
- New Jest test files under `apps/mobile/app/__tests__/` covering
