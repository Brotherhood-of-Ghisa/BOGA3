# t2: Drizzle schema additions + sync_runtime_state rewrite

**Type:** build

**Problem:** With v1 deleted (t1) the client-side schema needs the v2
shape: every entity table gains the two local-only sync columns from
t2 §9.1 (`local_dirty`, `local_updated_at_ms`); the five entity
tables missing `deleted_at` gain it per t1 §2 / §4.2; the
`sync_runtime_state` singleton is reshaped to carry the v2 columns
from t2 §9.2 / §9.3 (`pull_cursor`, `last_emitted_ms`,
`bootstrap_completed_at`, `applied_seed_migration_app_version`).
Because plan 2 assumes a **clean local DB on first v2 launch** (manual
wipe per `docs/plans/sync-v2-client/manual-wipe.md` — see t3), the
`sync_runtime_state` migration does NOT need to preserve any v1 rows
or column data. The migration may simply drop and recreate the table
in its v2 shape (or use Drizzle's standard 12-step swap if drift
checker / migration tooling prefers it) — there is no v1 row to
preserve because every device has been wiped. With these landed, the
drift checker shipped by plan 1 (`npm run check:sync-drift --
--strict`) must continue to pass, AND the `server_only_columns`
exemption registering `deleted_at` must be removed from
`apps/mobile/src/data/schema/sync-extras.json` (per `## Carry-over`
item 4 in `plan.md`) in the SAME PR — leaving it in place would mask
future legitimate-column regressions.

**Outcomes:**

- All eight entity Drizzle schemas declare `localDirty: integer('local_dirty',
  { mode: 'boolean' }).notNull().default(0)` and `localUpdatedAtMs:
  integer('local_updated_at_ms').notNull().default(0)`. Drizzle
  TypeScript camelCase; SQLite snake_case (per t2 §9.1 final paragraph).
- The five entity schemas that lacked one (`gyms.ts`,
  `session-exercises.ts`, `exercise-sets.ts`,
  `exercise-muscle-mappings.ts`, `session-exercise-tags.ts`) gain
  `deletedAt: integer('deleted_at', { mode: 'timestamp_ms' })`
  (nullable, no default) plus a matching `<table>_deleted_at_idx`
  index. The three already-present `deletedAt` columns
  (`sessions.ts`, `exercise-definitions.ts`,
  `exercise-tag-definitions.ts`) are unchanged.
- `apps/mobile/src/data/schema/sync-runtime-state.ts` is reshaped to
  the v2 shape. The reshaped schema declares:
  `pullCursor: text('pull_cursor', { mode: 'json' }).notNull().default('{}')`
  (storing a JSON object keyed by layer index `"0".."3"` per t2 §9.2),
  `lastEmittedMs: integer('last_emitted_ms').notNull().default(0)`,
  `bootstrapCompletedAt: integer('bootstrap_completed_at', { mode:
  'timestamp_ms' })` (nullable, preserved),
  `appliedSeedMigrationAppVersion: integer('applied_seed_migration_app_version').notNull().default(0)`,
  plus the existing primary-key `id` column. v1 columns
  (`is_enabled`, `bootstrap_user_id`, `last_bootstrap_error`,
  `last_bootstrap_attempt_at`, `seeds_applied_at`, `updated_at`) are
  dropped — the manual wipe means no v1 row data needs preservation,
  so the migration may drop and recreate the table directly, or use
  the standard SQLite 12-step swap if Drizzle's tooling produces that
  shape from the schema diff. The builder picks the simplest
  approach the drift checker accepts and surfaces the choice in the
  PR body.
- A new Drizzle migration registered in
  `apps/mobile/src/data/migrations/index.ts` implements every column
  add and the `sync_runtime_state` reshape. The generated migration
  SQL also lands under `apps/mobile/drizzle/` per `npm run db:generate`
  conventions.
- `apps/mobile/src/data/schema/sync-extras.json` no longer contains a
  `server_only_columns` exemption for `deleted_at`. The other
  exemptions (`local_only_columns` pair, `untyped_text_references`
  entry for `exercise_muscle_mappings.muscleGroupId`) are unchanged.
- `apps/mobile/src/data/exercise-catalog-seeds.ts`'s seed-marker
  read/write helpers reference
  `appliedSeedMigrationAppVersion` instead of `seedsAppliedAt`. The
  semantics shift from "timestamp" to "app-version integer";
  callers of these helpers (the seeder pipeline, the dev-reset
  helper) update accordingly so the behaviour ("re-seed when the
  catalog ships under a new app version, never overwrite
  user-edited rows") is preserved.
- `apps/mobile/src/data/dev-reset.ts` (`__clearSeedsAppliedMarkerForReset`)
  updates to use the renamed column.
- `npm run check:sync-drift -- --strict` exits zero with the
  `server_only_columns` exemption removed AND with the eight new
  `deleted_at` columns added (the three already-present plus the
  five added in this PR) — verified by the builder before opening
  the PR.
- `npm run lint && npm run typecheck && npm run test` passes from
  `apps/mobile/`.
- `test:e2e:ios:smoke` and `test:e2e:ios:data-smoke` pass locally;
  the PR body's Standard checklist asserts both with the built
  git sha.
