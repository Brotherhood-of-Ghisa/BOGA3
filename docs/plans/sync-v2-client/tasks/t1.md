# t1: delete v1 sync code paths

**Type:** build

**Problem:** The v1 client sync stack is the largest source of
incidental complexity in `apps/mobile/`: ~3000 lines under
`apps/mobile/src/sync/` plus `enqueueSyncEvent` / `enqueueSyncEventsTx`
call sites in every repo, plus the `sync_outbox_events` and
`sync_delivery_state` Drizzle tables, plus the wiring in
`apps/mobile/app/_layout.tsx`. The v2 design replaces the entire
event-log + outbox + projection model with two RPCs (push, pull) and
two local-only columns (`local_dirty`, `local_updated_at_ms`); none of
the v1 surface survives. Removing it cleanly BEFORE adding the v2
shape keeps every downstream PR's diff small and prevents the v1 + v2
codepaths from briefly coexisting in a confusing half-state. This
task is deletion-heavy (additions are limited to: a minimal
re-export shim if any non-sync caller still needs a type, and the
Drizzle migration that drops the two v1 tables).

**Inputs:**

- Plan `## Carry-over from plan 1` item 5: `topo-order.ts` survives.
- `apps/mobile/src/sync/` — all files except `topo-order.ts` are
  deleted.
- `apps/mobile/src/data/schema/sync-outbox-events.ts`,
  `apps/mobile/src/data/schema/sync-delivery-state.ts` — deleted from
  the Drizzle schema.
- `apps/mobile/src/data/schema/index.ts` — re-exports updated.
- `apps/mobile/src/data/migrations/index.ts` — a new migration (m0014
  is the next free idx; pick a `when` timestamp > the m0013
  `1779541826568`) drops the two v1 tables. SQLite `DROP TABLE IF
  EXISTS` is enough — the wipe in t3 will also delete the file on v2
  boot for existing devices.
- `apps/mobile/app/_layout.tsx` — remove the `startDefaultSyncScheduler`,
  `setDefaultSyncCadenceContextFromPathname`, `startSyncRuntime`,
  `stopSyncRuntime`, `stopDefaultSyncScheduler` calls. After t1 the
  layout has NO sync wiring; t7 adds the v2 scheduler bootstrap
  back in.
- Every repo under `apps/mobile/src/data/` that calls
  `enqueueSyncEvent` / `enqueueSyncEvents` / `enqueueSyncEventsTx`:
  `local-gyms.ts`, `session-drafts.ts`, `session-list.ts`,
  `exercise-catalog.ts`, `exercise-tags.ts` (and any others surfaced
  by Grep). t1 deletes the calls; t5a / t5b add the dirty-bit writes
  in their place.
- Every test under `apps/mobile/app/__tests__/` that exercises v1
  sync: `sync-bootstrap-*`, `sync-flush-partial-failure.test.ts`,
  `sync-outbox-engine.test.ts`, `sync-profile-status.test.ts`,
  `sync-reinstall-restore-parity.test.ts`, `sync-runtime-bootstrap.test.ts`,
  `sync-scheduler.test.ts`, `sync-domain-event-emission.test.ts`. These
  are DELETED. v2 tests land alongside t6 / t7 / tFINAL.

**Outcomes:**

- The eight v1 source files under `apps/mobile/src/sync/` are deleted:
  `engine.ts`, `outbox.ts`, `bootstrap.ts`, `runtime.ts`, `scheduler.ts`,
  `profile-status.ts`, `types.ts`, `index.ts`. The file
  `apps/mobile/src/sync/topo-order.ts` is preserved as-is.
- A NEW `apps/mobile/src/sync/index.ts` exists with zero exports (or
  is recreated only when t6 / t7 / t8 land their own modules; t1's
  delete leaves a stub `// intentionally empty — v2 modules land in
  later PRs of this plan` or removes the file).
- The Drizzle schema files
  `apps/mobile/src/data/schema/sync-outbox-events.ts` and
  `apps/mobile/src/data/schema/sync-delivery-state.ts` are deleted;
  the re-exports in `apps/mobile/src/data/schema/index.ts` are
  removed.
- A new Drizzle migration (idx 14, tag e.g.
  `0014_drop_v1_sync_tables`) executes
  `DROP TABLE IF EXISTS sync_outbox_events;` and
  `DROP TABLE IF EXISTS sync_delivery_state;` and is registered in
  `apps/mobile/src/data/migrations/index.ts`. The generated SQL file
  also lands under `apps/mobile/drizzle/` per the existing
  `db:generate` convention.
- Every repo file under `apps/mobile/src/data/` no longer imports
  any symbol from `@/src/sync/*`. The `enqueueSyncEvent` /
  `enqueueSyncEvents` / `enqueueSyncEventsTx` call sites are deleted;
  the surrounding transactional blocks (`database.transaction((tx) =>
  {...})`) remain in place — they will be revisited in t5a / t5b to
  add the `local_dirty` / `local_updated_at_ms` writes.
- `apps/mobile/app/_layout.tsx` no longer imports anything from
  `@/src/sync` and has no sync-related code in `useEffect`. The
  bootstrap call sequence (`bootstrapLocalDataLayer`,
  `bootstrapAuthState`, `ensureExerciseCatalogLoaded`) remains
  unchanged.
- Every v1 sync test under `apps/mobile/app/__tests__/` is deleted
  (see Inputs list); `is-dev-mode.test.ts` and unrelated tests
  remain.
- The constants `SYNC_BACKOFF_INITIAL_DELAY_MS`,
  `SYNC_BACKOFF_JITTER_RATIO`, `SYNC_BACKOFF_MAX_DELAY_MS`,
  `SYNC_BACKOFF_MULTIPLIER`, `SYNC_BATCH_MAX_SIZE`,
  `SYNC_SESSION_RECORDER_CADENCE_MS`, `SYNC_GENERAL_CADENCE_MS`,
  `SESSION_RECORDER_ROUTE_SEGMENT`, `BOOTSTRAP_COOLDOWN_MS`, and the
  function `syncCadenceContextFromPathname` no longer exist anywhere.
- `npm run lint && npm run typecheck && npm run test` passes from
  `apps/mobile/`.

**Output artifact:**

- Deletions: `apps/mobile/src/sync/{engine,outbox,bootstrap,runtime,scheduler,profile-status,types}.ts`
  (and `index.ts` either deleted or replaced with the empty stub
  above); `apps/mobile/src/data/schema/{sync-outbox-events,sync-delivery-state}.ts`;
  the eight v1 sync tests under `apps/mobile/app/__tests__/`.
- Additions: `apps/mobile/src/data/migrations/index.ts` gains m0014;
  `apps/mobile/drizzle/0014_*.sql` (a generated migration file
  whose exact filename comes from `npm run db:generate`).
- Edits: `apps/mobile/src/data/schema/index.ts` (remove the two
  re-exports); `apps/mobile/app/_layout.tsx` (remove sync wiring);
  every repo file under `apps/mobile/src/data/` that imported
  `@/src/sync/*` (delete the imports and the `enqueueSync*` call
  sites).

**Out of scope:**

- Adding `local_dirty` / `local_updated_at_ms` / `deleted_at` to
  entity Drizzle schemas (t2).
- Rewriting `sync_runtime_state` (t2).
- Adding back any new sync code (t4, t5a, t5b, t6, t7, t8, t9).
- The wipe of the on-device DB for v1→v2 migration (t3).
