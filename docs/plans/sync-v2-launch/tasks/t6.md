# t6: `muscle_groups` idempotent bootstrap

**Type:** build

**Problem:** `muscle_groups` is a client-only, non-syncable taxonomy seeded on
first launch. Today the seed gate is "all-or-nothing on empty": if the table
has any rows, no bundle muscle group is inserted, so a muscle group added to
the bundle in a later app version never reaches an existing device. The launch
contract extends the check to "insert any bundle row whose id is not already
present locally" so future taxonomy additions propagate. Muscle groups are
non-editable and client-only, so insert-if-not-exists carries no risk of
clobbering a user edit.

**Inputs:**
- t3 merged: the bootstrapper reorder is settled, so `muscle_groups` seeding
  stays decoupled at boot and t6 edits that path without colliding with the
  entity bootstrapper. (See t3 `## Output artifact`.)
- Binding design (authoritative — cite, do not restate):
  `docs/plans/sync-v2/designs/t3.md` §4 (muscle_groups is client-only,
  separately seeded, never on the wire), §4.4 + §8.2 #8 (extend the
  all-or-nothing check to per-id insert-if-not-exists:
  `FOR each muscle_group IN BUNDLE: IF NOT EXISTS(... WHERE id = mg.id): INSERT`).
- The muscle-group seed path: `apps/mobile/src/data/exercise-catalog-seeds.ts`
  (`SYSTEM_MUSCLE_GROUP_SEEDS`, the muscle-group portion of the seeder) and its
  invocation in `apps/mobile/src/data/bootstrap.ts`.
- `muscle_groups` schema: `apps/mobile/src/data/schema/muscle-groups.ts`
  (`is_editable = 0` guard; no `local_dirty` / `local_updated_at_ms` columns —
  it never syncs).

**Outcomes:**
- The muscle-group bootstrap inserts any bundle row whose id is not already
  present locally, on every launch (or app-version bump), instead of skipping
  all rows when the table is non-empty. (Maps to plan outcome: `muscle_groups`
  bootstrap extended from all-or-nothing to insert-if-not-exists.)
- Existing local muscle-group rows are not overwritten or duplicated; ids are
  the join key. `is_editable` stays 0 on seeded rows.
- Muscle groups still never carry sync-bookkeeping columns and never cross the
  wire.
- Jest covers: a fresh table seeds the full bundle; a table missing one bundle
  id gains exactly that row on the next run; a fully-seeded table is a no-op; a
  pre-existing row is not overwritten.
- Quality gate `./scripts/quality-fast.sh frontend` passes.

**Output artifact:**
- The updated muscle-group seed routine in
  `apps/mobile/src/data/exercise-catalog-seeds.ts` (insert-if-not-exists by id).
- Jest specs under `apps/mobile/app/__tests__/`.

**Out of scope:**
- Any change to the syncable entity bootstrapper (t3) or the entity seeder.
- Promoting `muscle_groups` to a syncable entity (explicitly out of scope per
  design t3.md §4.3).
- The bundle-migration loop (t5).
