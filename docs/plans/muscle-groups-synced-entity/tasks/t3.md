# t3: client schema + single-baseline regen

**Type:** build

> Updated from t1: see designs/t1.md ## Decision

**Problem:** The client `muscle_groups` Drizzle schema lacks the sync-bookkeeping columns
(`deletedAt`, `localDirty`, `localUpdatedAtMs`) and an `id` default, and it still carries the
`muscle_groups_non_editable_guard` CHECK that would block the entity from being a normal synced
table. The single squashed `m0000` client baseline must be regenerated to reflect the new shape
while keeping exactly one journal entry keyed `m0000` (tag `0000_living_bucky`) — no incremental
client migration, fresh-installs-only.

**Inputs:** t1 merged; binding contract in `designs/t1.md ## Decision` (item 1 — the final client
column set). Key files:
- `apps/mobile/src/data/schema/muscle-groups.ts` — the schema to amend.
- `apps/mobile/src/data/schema/exercise-definitions.ts` — the template (sync columns, `id` default).
- `apps/mobile/src/data/schema/exercise-muscle-mappings.ts` — keep the FK
  `muscleGroupId → muscleGroups.id`.
- Regeneration command: `npm run db:generate` (drizzle-kit generate + `scripts/bundle-migrations.ts`),
  run from `apps/mobile`. Generated artifacts: `apps/mobile/drizzle/0000_living_bucky.sql`,
  `apps/mobile/drizzle/meta/0000_snapshot.json`, `apps/mobile/drizzle/migrations.generated.ts`.
- Invariant guard: `apps/mobile/app/__tests__/domain-schema-migrations.test.ts` (asserts
  `localRuntimeMigrations.journal.entries` length 1 and keys `['m0000']`).

**Outcomes (maps to plan outcomes 3, 4):**
- `apps/mobile/src/data/schema/muscle-groups.ts` matches `designs/t1.md ## Decision` item 1: adds
  `deletedAt` (timestamp_ms, nullable), `localDirty` (boolean, notNull default false),
  `localUpdatedAtMs` (integer, notNull default 0); adds the `id` default
  `lower(hex(randomblob(16)))`; keeps `displayName` / `familyName` / `sortOrder` / `isEditable` /
  `createdAt` / `updatedAt`; DROPS the `muscle_groups_non_editable_guard` CHECK; keeps the boolean
  guard and the `sort_order >= 0` guard; keeps the declared indexes.
- `apps/mobile/src/data/schema/exercise-muscle-mappings.ts` still declares the client FK
  `muscleGroupId → muscleGroups.id`.
- `npm run db:generate` was run; the three generated artifacts reflect the new `muscle_groups` shape
  and the dropped guard, with the `0000_living_bucky` tag preserved.
- `localRuntimeMigrations.journal.entries` has length 1 with keys `['m0000']`;
  `domain-schema-migrations.test.ts` is green.
- `./scripts/quality-fast.sh` and `npm run test:handles` are green. Evidence in the PR.

**Output artifact:** `apps/mobile/src/data/schema/muscle-groups.ts` (amended), the three regenerated
files `apps/mobile/drizzle/0000_living_bucky.sql`, `apps/mobile/drizzle/meta/0000_snapshot.json`,
`apps/mobile/drizzle/migrations.generated.ts`, and the preserved FK in
`apps/mobile/src/data/schema/exercise-muscle-mappings.ts`.

**Out of scope:** Sync-registry edits (`topo-order.ts`, `cycle.ts` — t4). Seeding / bootstrap (t5).
Any incremental client migration or new journal entry. Editing existing-device upgrade paths.
