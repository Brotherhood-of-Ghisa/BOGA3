**Problem:** `muscle_groups` is currently seeded as a standalone every-boot additive taxonomy
(`seedMuscleGroups` via `seedBootDataLayer`) and is treated as never-synced by bootstrap and
account-wipe. As a synced entity it must instead be seeded `local_dirty = 1` through the same
version-marker-gated starter-catalog path as `exercise_definitions` (`seedSystemExerciseCatalog`),
removed from the boot-only seed, and wiped/recovered generically. Separately, FK enforcement
(`PRAGMA foreign_keys = ON`) must be enabled at boot — after migrations, before seeding — so the FK
is live in production; the pragma is a no-op inside a transaction and table-rebuild migrations need
it off, so it must be set post-`migrate()`.
