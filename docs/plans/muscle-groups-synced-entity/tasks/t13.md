# t13: remove durable plan-id leak in bundle-migrations.ts

**Type:** build

**Problem:** A plan/card identifier leaked into a durable source file. The plan's
own **Durable-artifact hygiene** rule (plan.md `## Orchestration`) and the
orchestration protocol both require that source, comments, test names, and commit
messages NEVER reference plan/card ids or `docs/plans/...` paths — the plan dir is
deleted on landing, so such a reference becomes a dangling, meaningless token in
permanent history. The audit found exactly one violation across the whole repo:

`apps/mobile/scripts/bundle-migrations.ts:12`
```
 * Pre-t10 we hand-copied the SQL string into
```

`t10` is a plan card id. The comment's intent is "before this script existed we
hand-copied the SQL"; it must say that in self-contained terms with no card id.
t10 (PR #176) was itself the hygiene-sweep task, but its sweep covered `supabase/`
and missed this `apps/mobile/scripts/` file.

**Inputs:** none (independent one-line doc-comment fix). Branch off the latest
`origin/main`.

**Outcomes:**
- `apps/mobile/scripts/bundle-migrations.ts:12` no longer contains `t10` (or any
  `t<n>` / `tFINAL` / `docs/plans/...` token). The reworded comment is
  self-contained and preserves the original meaning (e.g. "Before this script
  existed, the SQL string was hand-copied into …").
- A repo-wide grep for durable plan-id leaks is clean:
  `grep -rniE '\bt[0-9]+\b|tFINAL|docs/plans/muscle' apps/mobile/src apps/mobile/app apps/mobile/scripts apps/mobile/drizzle supabase/migrations supabase/tests supabase/scripts docs/specs RUNBOOK.md`
  returns no plan/card-id reference (the `\bt[0-9]+\b` matches must all be
  unrelated tokens, not card ids).
- No functional change: the generated bundle output is byte-identical
  (`npm run db:generate` produces no diff). Comment-only edit.
- `./scripts/quality-fast.sh` and `npm run test:handles` are green. Evidence in
  the PR. The backend and frontend slow lanes are ⛔ N/A (comment-only change to a
  build-time script with no runtime or schema effect — state the N/A reason in the
  PR Tests section).

**Output artifact:** `apps/mobile/scripts/bundle-migrations.ts` (line ~12,
comment reworded).

**Out of scope:** Any behavior change to the migration bundler. Touching the
generated `apps/mobile/drizzle/migrations.generated.ts`. Reformatting unrelated
comment lines. Any other file (this is the sole leak the audit found).
