# t11: cross-owner RLS contract coverage for muscle_groups

**Type:** build

> Updated from t1: see designs/t1.md ## Decision (the four owner RLS policies on
> `app_public.muscle_groups`).

**Problem:** The backend cross-owner RLS contract test asserts, for each per-user entity table, that
owner B cannot select / insert-as / update / delete owner A's rows (the four-policy owner-isolation
guarantee). It enumerates the entities explicitly and was never extended to the new
`app_public.muscle_groups` table. The four owner RLS policies DO exist on the server (added with the
table) and the drift checker structurally hash-verifies them, but no RUNTIME probe confirms
`muscle_groups` actually enforces cross-owner isolation. For parity with the other eight entities —
and because RLS isolation is a security guarantee — `muscle_groups` must be probed too.

**Inputs:** t2 merged (the `app_public.muscle_groups` table + its four owner RLS policies + grants)
and t10 merged (the backend contract suites are hygienic and the full backend lane is green on a
clean reset). Binding contract: `designs/t1.md ## Decision` (item 2 — the RLS policy set). Key file:
- `supabase/tests/sync-v2-rls-cross-owner.sh` — add `muscle_groups` to the set of entities probed for
  cross-owner isolation, mirroring a sibling Layer-0 entity (e.g. `exercise_definitions`): seed a
  `muscle_groups` row owned by user A, then assert user B's role cannot read it, cannot update it,
  cannot delete it, and cannot insert a row claiming A's `owner_user_id`.

**Outcomes:**
- `sync-v2-rls-cross-owner.sh` exercises `muscle_groups` cross-owner isolation with the same
  select / insert / update / delete probes it applies to the other entities; the new entity is no
  longer skipped.
- The probe passes against the live server schema (the four owner policies enforce isolation).
- `./scripts/quality-slow.sh backend` is green end-to-end on a clean reset (this suite plus the
  existing ones). `./scripts/quality-fast.sh` and `npm run test:handles` are green.

**Output artifact:** `supabase/tests/sync-v2-rls-cross-owner.sh` extended to cover `muscle_groups`.

**Out of scope:** Client code, seeding/bootstrap (t5), client FK tests (t7), docs (t8), the drift
checker (t6 — already covers RLS structurally). Adding RLS probes for entities other than
`muscle_groups`. Any change to the RLS policies themselves (they already exist and are correct).
