# t6: drift checker — drop muscleGroupId exemption, 9th entity

**Type:** build

**Problem:** The schema-drift checker exempts `exercise_muscle_mappings.muscleGroupId` from the
typed-server-column rule via `sync-extras.json`'s `untyped_text_references`, because `muscle_groups`
used to be client-only. Now that `muscle_groups` is a real server table with a typed
`muscle_group_id` FK, that exemption would mask drift and must be removed, and the checker must treat
`muscle_groups` as the 9th entity table. The checker derives `ENTITY_TABLES` from the live schema
(every `app_public` table with `owner_user_id`), so it should pick up the new table automatically —
but any hardcoded "8"/entity-count expectations must be updated, and the topo assertion must accept
`muscle_groups` in Layer 0.

**Inputs:** t4 merged (registry has `muscle_groups` in Layer 0). Binding contract in
`designs/t1.md ## Decision` (items 2-5). Key files:
- `apps/mobile/src/data/schema/sync-extras.json` — remove the `untyped_text_references` entry for
  `muscleGroupId` (and the `untyped_text_references` key if it becomes empty).
- `apps/mobile/scripts/check-sync-schema-drift.ts` — confirm the `untyped_text_references` handling
  (~lines 628, 722, 863) no longer needs the muscle-group special case; update any "8"/entity-count
  assumptions so 9 entity tables pass; ensure the topo FK-order assertion accepts `muscle_groups` in
  Layer 0 and the `muscle_group_id → muscle_groups` FK as a valid Layer 1 → Layer 0 edge.
- `apps/mobile/app/__tests__/sync/drift-check.test.ts` — the infra-lane assertion that the checker
  exits 0 under `--strict`; update for 9 entities + the new FK.
- `apps/mobile/app/__tests__/sync/drift-exemption-removed.test.ts` — extend (or mirror the existing
  pattern) to assert the `muscleGroupId` `untyped_text_references` exemption is absent from
  `sync-extras.json`.

**Outcomes (maps to plan outcome 7):**
- `sync-extras.json` no longer carries the `muscleGroupId` `untyped_text_references` exemption.
- `check-sync-schema-drift.ts` treats `muscle_groups` as a normal entity table: derives 9 entities
  from the live schema (no hardcoded "8" remains), validates the `muscle_group_id` typed column and
  FK, and its topo assertion accepts `muscle_groups` in Layer 0.
- The checker exits 0 under `--strict` against the t2 server schema (run as part of
  `./scripts/quality-slow.sh backend`).
- `drift-exemption-removed.test.ts` asserts the exemption removal (fast lane);
  `drift-check.test.ts` reflects the 9-entity expectation.
- `./scripts/quality-fast.sh`, `npm run test:handles`, and `./scripts/quality-slow.sh backend` are
  green. Evidence in the PR.

**Output artifact:** `apps/mobile/src/data/schema/sync-extras.json` (exemption removed),
`apps/mobile/scripts/check-sync-schema-drift.ts` (9-entity / topo updates),
`apps/mobile/app/__tests__/sync/drift-exemption-removed.test.ts` and
`apps/mobile/app/__tests__/sync/drift-check.test.ts` (updated assertions).

**Out of scope:** Server DDL/RPC (t2 — this card consumes it). Client schema (t3). Registry (t4).
Seeding / bootstrap (t5). Round-trip / FK-harness tests (t7). Docs (t8).
