# t4: sync registry — topo-order + cycle field/table maps

**Type:** build

**Problem:** The sync engine's entity registry is hardcoded to the eight current entities.
`muscle_groups` must be registered as the 9th: added to `EntityTableName` and Layer 0 of
`TOPO_LAYERS`, and given an `ENTITY_FIELDS` wire-column list and an `ENTITY_TABLES` Drizzle-table
binding. Without this the push/pull cycle cannot serialize or apply `muscle_groups` rows.

**Inputs:** t1 merged and t3 merged. Binding contract in `designs/t1.md ## Decision` (item 4 — the
wire field map; item 5 — Layer 0 placement). t3 provides the client schema symbol
`schema.muscleGroups` with its sync columns. Key files:
- `apps/mobile/src/sync/topo-order.ts` — `TOPO_LAYERS` (add `'muscle_groups'` to Layer 0) and the
  layering property docs; `EntityTableName` is derived from `TOPO_LAYERS` so it updates
  automatically — confirm the type widens to include `'muscle_groups'`.
- `apps/mobile/src/sync/cycle.ts` — `ENTITY_FIELDS` (add the `muscle_groups` entry; `exercise_definitions`
  entry ~line 200 is the template; use the `TS`/`SC` field specs per the wire field map) and
  `ENTITY_TABLES` (add `muscle_groups: schema.muscleGroups`; ~line 262).

**Outcomes (maps to plan outcome 5):**
- `'muscle_groups'` is a member of `EntityTableName` and appears in **Layer 0** of `TOPO_LAYERS`
  alongside `gyms` and `exercise_definitions`; the layer property docs mention it.
- `ENTITY_FIELDS['muscle_groups']` lists exactly the wire columns from `designs/t1.md ## Decision`
  item 4: `display_name, family_name, sort_order, is_editable, created_at, updated_at, deleted_at`
  with the correct `TS`/`SC` field-spec kinds.
- `ENTITY_TABLES['muscle_groups']` binds to `schema.muscleGroups`.
- The `Record<EntityTableName, …>` maps remain exhaustive (TypeScript compiles), confirming all nine
  entities are present.
- `./scripts/quality-fast.sh` and `npm run test:handles` are green. Evidence in the PR.

**Output artifact:** `apps/mobile/src/sync/topo-order.ts` (`muscle_groups` in Layer 0, updated docs)
and `apps/mobile/src/sync/cycle.ts` (`ENTITY_FIELDS['muscle_groups']`,
`ENTITY_TABLES['muscle_groups']`).

**Out of scope:** Server DDL/RPC (t2). Seeding / bootstrap (t5). Drift checker (t6). Tests (t7) —
